# ДИЗАЙН ИНТЕГРАЦИИ CRM В МЕССЕНДЖЕР

## Концепция: "Мессенджер + CRM = Единая платформа"

## ГИБРИДНАЯ АРХИТЕКТУРА

CRM работает в ДВУХ режимах одновременно:

### Режим 1: Отдельный сервис (ServicesMenu)
```
┌─────────────────────────────────────────────────┐
│  ServicesMenu (при тапе на логотип)            │
│                                                 │
│   ┌────────┐ ┌────────┐ ┌────────┐           │
│   │  Такси │ │Каршер. │ │Доставка│           │
│   └────────┘ └────────┘ └────────┘           │
│                                                 │
│   ┌────────┐ ┌────────┐ ┌────────┐           │
│   │Недвиж.│ │Страхов.│ │  CRM   │ ← Тап сюда │
│   └────────┘ └────────┘ └────────┘           │
│                                                 │
│   → Открывается /crm (полноэкранный режим)    │
└─────────────────────────────────────────────────┘
```

### Режим 2: Встроенная панель в чате
```
┌───────────────────────────────────┐ ┌─────────────────────┐
│  Чат                              │ │ 💼 CRM Панель      │
│                                   │ │                     │
│  ┌─────────────────────────────┐ │ │ 👤 Клиент          │
│  │ Сообщения...                │ │ │ Сделка             │
│  │                             │ │ │ Задачи             │
│  └─────────────────────────────┘ │ │                     │
│                                   │ │ [📝 Открыть /crm]  │
│                                   │ └─────────────────────┘
└───────────────────────────────────┘
```

### Связь между режимами
- Данные синхронизируются в реальном времени
- Клиент созданный в чате → виден в /crm
- Сделка из /crm → привязывается к чату с клиентом
- Задачи создаются из любого режима


### Как это работает:

1. **Из чата в CRM:**
   - Клик по кнопке "💼 CRM" → открывается CRM панель
   - Автоматически показывает данные текущего клиента
   - Можно создать сделку/задачу не выходя из чата

2. **Из CRM в чат:**
   - Клик по кнопке "💬 Чат" → открывается чат с выбранным клиентом
   - Можно написать сообщение клиенту напрямую

3. **Полноэкранный режим:**
   - Тап на "CRM" в ServicesMenu → полноэкранный /crm
   - Работает как самостоятельное приложение внутри мессенджера

## 8. ТЕХНИЧЕСКАЯ РЕАЛИЗАЦИЯ

### 8.1 Структура роутов

```
/crm                    → Главная страница CRM (выбор профессии)
/crm/dashboard         → Дашборд (воронка, статистика)
/crm/clients           → Список клиентов
/crm/clients/:id      → Карточка клиента
/crm/deals             → Список сделок
/crm/deals/:id         → Карточка сделки
/crm/tasks             → Задачи
/crm/auto              → Автомобильный модуль
/crm/real-estate       → Недвижимость
/crm/hr                → HR модуль
/crm/smm               → SMM модуль
/crm/finance           → Финансы
```

### 8.3 База данных (дополнительные таблицы)

```sql
-- Профессиональный профиль пользователя
CREATE TABLE crm.user_profiles (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES auth.users,
    profession_code VARCHAR(50),  -- 'AUTO_DEALER', 'REAL_ESTATE_AGENT', etc.
    settings JSONB,              -- Настройки под профессию
    created_at TIMESTAMPTZ
);

-- Клиенты (универсальные)
CREATE TABLE crm.clients (
    id UUID PRIMARY KEY,
    owner_id UUID REFERENCES auth.users,
    name VARCHAR(255),
    phone VARCHAR(50),
    email VARCHAR(255),
    profession_data JSONB,  -- Данные специфичные для профессии
    created_at TIMESTAMPTZ
);

-- Сделки
CREATE TABLE crm.deals (
    id UUID PRIMARY KEY,
    owner_id UUID REFERENCES auth.users,
    client_id UUID REFERENCES crm.clients,
    title VARCHAR(255),
    amount DECIMAL(15,2),
    stage VARCHAR(50),       -- 'NEW', 'NEGOTIATION', 'BOOKING', 'DONE'
    probability INT,
    expected_close_date DATE,
    created_at TIMESTAMPTZ
);

-- Задачи
CREATE TABLE crm.tasks (
    id UUID PRIMARY KEY,
    owner_id UUID REFERENCES auth.users,
    deal_id UUID REFERENCES crm.deals,
    client_id UUID REFERENCES crm.clients,
    title VARCHAR(255),
    due_date DATE,
    completed BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ
);
```

## 11. ПРЕИМУЩЕСТВА ГИБРИДНОГО ПОДХОДА

| Режим | Когда использовать |
|-------|-------------------|
| **CRM в чате** | Быстрые действия, не выходя из переписки |
| **Полноэкранный CRM** | Глубокая работа, аналитика, управление |
| **ServicesMenu** | Быстрый доступ к CRM издалека |

### Сравнение с конкурентами:

| Функция | Обычные CRM | Наша система (ГИБРИД) |
|---------|-------------|----------------------|
| Интеграция с чатом | Отдельное приложение | Всё в одном |
| CRM встроенная панель | Редко | Да (гибрид) |
| AI помощник | Плагин | Встроен в интерфейс |
| Выбор профессии | Настройка вручную | Автоматически |
| Оффлайн режим | Ограниченно | Полная поддержка |
| Стоимость | 1000+ руб/мес | Включено в мессенджер |
| Два режима работы | Нет | Да (сервис + в чате) |

---

Этот дизайн показывает, как CRM интегрируется в существующий мессенджер вГИБРИДНОМ режиме - и как отдельный сервис, и как встроенная панель в чате, сохраняя единый пользовательский опыт и синхронизацию данных.

## 14. ТЕХНИЧЕСКАЯ РЕАЛИЗАЦИЯ (ДЕТАЛЬНО)

### 14.1 База данных - Полная схема

```sql
-- Схема CRM (в схеме crm)

-- Профессиональный профиль пользователя
CREATE TABLE crm.user_profiles (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    profession_code VARCHAR(100) NOT NULL,
    business_name VARCHAR(255),
    settings JSONB DEFAULT '{}',
    subscription_tier VARCHAR(50) DEFAULT 'free',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Универсальные клиенты
CREATE TABLE crm.clients (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    owner_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    name VARCHAR(255) NOT NULL,
    phone VARCHAR(50),
    email VARCHAR(255),
    telegram_id BIGINT,
    address TEXT,
    notes TEXT,
    tags TEXT[],
    avatar_url TEXT,
    profession_data JSONB DEFAULT '{}',
    custom_fields JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Сделки
CREATE TABLE crm.deals (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    owner_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    client_id UUID REFERENCES crm.clients(id) ON DELETE SET NULL,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    amount DECIMAL(15,2),
    currency VARCHAR(10) DEFAULT 'RUB',
    stage VARCHAR(50) NOT NULL DEFAULT 'NEW',
    probability INT DEFAULT 10,
    expected_close_date DATE,
    actual_close_date DATE,
    deal_type VARCHAR(50),
    products JSONB DEFAULT '[]',
    custom_fields JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Этапы сделок
CREATE TABLE crm.deal_stages (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    owner_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,
    order_index INT NOT NULL DEFAULT 0,
    probability INT DEFAULT 0,
    color VARCHAR(20),
    is_won BOOLEAN DEFAULT FALSE,
    is_lost BOOLEAN DEFAULT FALSE
);

-- Задачи
CREATE TABLE crm.tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    owner_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    deal_id UUID REFERENCES crm.deals(id) ON DELETE SET NULL,
    client_id UUID REFERENCES crm.clients(id) ON DELETE SET NULL,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    due_date TIMESTAMPTZ,
    completed_at TIMESTAMPTZ,
    priority VARCHAR(20) DEFAULT 'normal',
    task_type VARCHAR(50),
    reminder_at TIMESTAMPTZ,
    custom_fields JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Взаимодействия (звонки, встречи, заметки)
CREATE TABLE crm.interactions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    owner_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    client_id UUID REFERENCES crm.clients(id) ON DELETE SET NULL,
    deal_id UUID REFERENCES crm.deals(id) ON DELETE SET NULL,
    type VARCHAR(50) NOT NULL,
    direction VARCHAR(20),
    content TEXT,
    duration INT,
    result VARCHAR(100),
    next_action TEXT,
    custom_fields JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Связь чатов с клиентами
CREATE TABLE crm.chat_client_links (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    chat_id UUID NOT NULL,
    client_id UUID REFERENCES crm.clients(id) ON DELETE CASCADE,
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    linked_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(chat_id, user_id)
);

-- Шаблоны документов
CREATE TABLE crm.document_templates (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    owner_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    name VARCHAR(255) NOT NULL,
    type VARCHAR(50) NOT NULL,
    content TEXT NOT NULL,
    variables JSONB DEFAULT '[]',
    profession_code VARCHAR(100),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Специфичные данные для автобизнеса
CREATE TABLE crm.auto_data (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    client_id UUID REFERENCES crm.clients(id) ON DELETE CASCADE,
    vehicle_mark VARCHAR(100),
    vehicle_model VARCHAR(100),
    vehicle_year INT,
    vehicle_vin VARCHAR(50),
    vehicle_price DECIMAL(15,2),
    vehicle_status VARCHAR(50),
    test_drive_booked TIMESTAMPTZ,
    insurance_policy VARCHAR(100),
    credit_approved BOOLEAN DEFAULT FALSE,
    custom_fields JSONB DEFAULT '{}'
);

-- Специфичные данные для недвижимости
CREATE TABLE crm.real_estate_data (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    client_id UUID REFERENCES crm.clients(id) ON DELETE CASCADE,
    property_type VARCHAR(50),
    property_address TEXT,
    property_area DECIMAL(10,2),
    property_rooms INT,
    property_floor INT,
    property_price DECIMAL(15,2),
    property_status VARCHAR(50),
    property_photos TEXT[],
    showings JSONB DEFAULT '[]',
    custom_fields JSONB DEFAULT '{}'
);
```

```sql
-- Клиенты: пользователь видит только свои данные
CREATE POLICY "Users can view own clients" ON crm.clients
  FOR SELECT USING (owner_id = auth.uid());

CREATE POLICY "Users can insert own clients" ON crm.clients
  FOR INSERT WITH CHECK (owner_id = auth.uid());

CREATE POLICY "Users can update own clients" ON crm.clients
  FOR UPDATE USING (owner_id = auth.uid());

-- Сделки: + видимость для админов
CREATE POLICY "Manager can view all deals" ON crm.deals
  FOR SELECT USING (
    owner_id = auth.uid() 
    OR EXISTS (SELECT 1 FROM crm.user_profiles 
               WHERE user_id = auth.uid() AND profession_code = 'ADMIN')
  );

-- Аудит изменений
CREATE TABLE crm.audit_log (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES auth.users(id),
    action VARCHAR(50) NOT NULL,
    table_name VARCHAR(100),
    record_id UUID,
    changes JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```
