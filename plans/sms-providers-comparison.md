## Детальное сравнение для России

### Twilio (рекомендуемый)
```
Цена: $0.04-0.08 за SMS в Россию
API: RESTful, простой
Документация: ⭐⭐⭐⭐⭐
Доставка: Хорошая
Регистрация: Требуется бизнес-верификация
```

**Плюсы:**
- Надежный, высокая доставка
- Много функций
- Webhook для статусов

**Минусы:**
- Дороже некоторых
- Требует бизнес-верификацию

### Plivo (самый дешевый)
```
Цена: $0.02-0.04 за SMS в Россию
API: RESTful, совместим с Twilio
Документация: ⭐⭐⭐⭐
Доставка: Средняя
Регистрация: Простая
```

**Плюсы:**
- Дешевле Twilio в 2 раза
- Простая интеграция
- Есть бесплатные кредиты

**Минусы:**
- Менее известен
- Поддержка хуже

### SMS77 (для Германии/ЕС)
```
Цена: €0.02-0.04 за SMS
API: RESTful
Документация: ⭐⭐⭐⭐
Доставка: Хорошая в EU
Регистрация: Быстрая
```

**Плюсы:**
- Дешевый для EU
- Немецкое качество

**Минусы:**
- Не оптимизирован для РФ

### MessageBird
```
Цена: €0.04-0.08 за SMS
API: RESTful
Документация: ⭐⭐⭐⭐⭐
Доставка: Отличная
Регистрация: Требуется
```

**Плюсы:**
- Очень надежный
- Много функций
- Хорошая аналитика

**Минусы:**
- Дорогой

## Рекомендация

### Для проекта (баланс цена/качество):

1. **Plivo** - самый дешевый, хорошее качество
2. **Twilio** - если нужна максимальная надежность
3. **MessageBird** - если нужны дополнительные функции

### Интеграция в проект

Сейчас в проекте используется SMS.ru в файле:
`supabase/functions/send-sms-otp/index.ts`

Для перехода на новый провайдер нужно:
1. Зарегистрироваться на Plivo/Twilio
2. Получить API ключи
3. Заменить HTTP-запрос к SMS.ru на нового провайдера
4. Обновить .env переменные
