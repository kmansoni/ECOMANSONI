# План: Собственный сервис отправки SMS/Push

## Текущая ситуация
- Проект использует **SMS.ru API** для отправки OTP-кодов
- Зависимость от стороннего сервиса (sms.ru)
- Файл: `supabase/functions/send-sms-otp/index.ts`

## Цель
Создать собственный сервис для коммуникации с пользователями без зависимости от third-party SMS провайдеров.

## Варианты реализации

### Вариант 1: Push-уведомления (РЕКОМЕНДУЕМЫЙ)
**Стоимость:** Бесплатно  
**Зависимость:** Нет  
**Сложность:** Средняя

```
┌─────────────┐     ┌──────────────┐     ┌─────────────────┐
│   Frontend  │────▶│  Your Server │────▶│   FCM (Google)  │
│  (Mobile)   │◀────│              │◀────│  (Free Push API)│
└─────────────┘     └──────────────┘     └─────────────────┘
```

**Преимущества:**
- Бесплатно (Google Firebase бесплатен до лимитов)
- Мгновенная доставка
- Не требует SIM/модема
- Работает везде, где есть интернет

**Недостатки:**
- Требует установки приложения на телефон клиента
- Не работает как традиционное SMS на простые телефоны

### Вариант 2: GSM-модем
**Стоимость:** ~2000 руб (硬件)  
**Зависимость:** Нет  
**Сложность:** Средняя

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│   Server    │────▶│  GSM-модем   │────▶│   SMSC      │
│             │     │  (Huawei)    │     │  (Оператор) │
└─────────────┘     └─────────────┘     └─────────────┘
```

**Преимущества:**
- Полный контроль
- Работает без интернета
- Настоящие SMS

**Недостатки:**
- Требует SIM-карту с SMS-пакетом
- Низкая скорость (10-50 SMS/час)
- Оборудование может сломаться

### Вариант 3: Kannel (Open Source SMS Gateway)
**Стоимость:** Бесплатно ( ПО)  
**Зависимость:** Нет  
**Сложность:** Высокая

Аналогично Варианту 2, но использует профессиональный open-source шлюз.

## Важное примечание

⚠️ **Технически невозможно** отправить SMS без:
- Договора с оператором связи
- SIM-карты с SMS-тарифом
- Подключения к SMSC оператора

Альтернативы:
1. **Push** - через интернет (бесплатно)
2. **Email-to-SMS** - некоторые операторы поддерживают (ограничено)
3. **GSM-модем** - свой "мини-оператор"

Для данного проекта (AI-компаньон мессенджер) **Push - оптимальный выбор**.
