## 1. Uber

**Страна:** США | **Охват:** 70+ стран, 10 000+ городов | **Основан:** 2009

### A. Архитектура системы

#### Тип архитектуры: Микросервисная — более 4000 микросервисов

Uber прошёл путь от монолитного Python-приложения (2010–2014) к распределённой системе из тысяч микросервисов.

**Эволюция архитектуры:**

| Период | Архитектура | Стек |
|--------|-------------|------|
| 2010–2012 | Монолит | Python, SQLAlchemy, PostgreSQL |
| 2013–2015 | SOA | Python, Java, Go, Riak, Redis |
| 2016–н.в. | Микросервисная | Java, Go, Node.js, Python |

**Ключевые сервисы и их взаимодействие:**

- **Dispatch Service — DISCO** — центральный сервис матчинга водителей и пассажиров. Использует геопространственный индекс на основе Google S2 library.
- **Trip Service** — управление жизненным циклом поездки от создания до завершения.
- **Pricing Service — Surge** — расчёт динамического ценообразования на основе спроса и предложения.
- **Payment Service** — обработка платежей, интеграция с платёжными шлюзами.
- **Location Service** — обработка GPS-данных от миллионов устройств в реальном времени.
- **Maps Service** — собственная картографическая платформа, маршрутизация, ETA.
- **User Service** — управление профилями пассажиров и водителей.
- **Notification Service** — push-уведомления, SMS, email.
- **Fraud Detection Service** — антифрод на базе ML-моделей.
- **Marketplace Service** — балансировка спроса и предложения.

**Системы обмена сообщениями:**

- **Apache Kafka** — основная шина событий, обрабатывает триллионы сообщений в день. Uber — крупнейший пользователь Kafka в мире.
- **gRPC** — синхронное взаимодействие между сервисами, заменил Apache Thrift.
- **Cherami** — собственная система очередей с открытым исходным кодом, позже заменена на Kafka.
- **Ringpop** — consistent hashing для маршрутизации запросов к stateful-сервисам.
- **TChannel** — собственный RPC-протокол, позже заменён на gRPC.

**Подход к хранению данных:**

| Технология | Назначение |
|-----------|-----------|
| MySQL — Schemaless | Основное хранилище с собственной надстройкой для schemaless-доступа и шардирования |
| PostgreSQL | Аналитика, геоданные через PostGIS |
| Apache Cassandra | Высоконагруженные write-heavy сценарии — логи, события |
| Redis | Кеширование, сессии, геопространственные индексы |
| Apache Hive и Presto | Аналитическое хранилище — data lake |
| Elasticsearch | Полнотекстовый поиск, логирование |
| Apache Pinot | Real-time аналитика — замена Apache Druid |
| Docstore | Собственное документоориентированное хранилище на базе MySQL |

**Инфраструктура:**

- Миграция с AWS на Google Cloud Platform в 2017–2018 годах.
- **Peloton** — собственный оркестратор поверх Apache Mesos, позже миграция на Kubernetes.
- **Docker** — контейнеризация всех сервисов.
- **Jaeger** — собственная система распределённой трассировки, передана в CNCF.
- **M3** — собственная платформа метрик на основе Prometheus.
- **Cadence и Temporal** — оркестрация долгоживущих workflow.

> **Источники:** [Uber Engineering Blog](https://eng.uber.com/), [InfoQ: Uber Architecture](https://www.infoq.com/presentations/uber-microservices/)

### C. Ключевые подсистемы

#### Алгоритм матчинга — DISCO

- Использует **Google S2 Geometry Library** для разбиения поверхности Земли на ячейки
- Водители индексируются по S2-ячейкам, обновление позиции каждые 4 секунды
- **Батчевый матчинг** — запросы пассажиров собираются в окна по 200 мс и решаются как задача оптимизации
- Алгоритм учитывает: расстояние, ETA, направление движения водителя, тип автомобиля, рейтинги, совместимость для Pool
- Используется **Hungarian Algorithm** для оптимального назначения в батче

> **Источник:** [Uber Engineering — Fulfillment Platform](https://eng.uber.com/fulfillment-platform-rearchitecture/)

#### Система динамического ценообразования — Surge Pricing

- Город разбивается на гексагональные зоны через **H3** — собственную геоиндексную систему
- В каждой зоне в реальном времени рассчитывается соотношение спроса и предложения
- Коэффициент surge рассчитывается ML-моделью с учётом: текущего спроса, количества водителей, погоды, событий, времени суток, исторических паттернов
- Surge отображается на карте пассажира как мультипликатор — 1.2x, 1.5x, 2.0x
- Водители видят зоны повышенного спроса на тепловой карте

> **Источник:** [Uber H3](https://h3geo.org/)

#### Маршрутизация и навигация

- Собственный маршрутизатор на основе **OSRM**, значительно доработанный
- Учитывает реальный трафик, исторические данные, ограничения, тип транспорта
- **ETA Engine** — предсказание времени прибытия на основе ML с точностью 1–2 минуты
- Карты: собственная платформа на основе OpenStreetMap с проприетарными данными

#### Платёжная система

- Поддерживает 400+ методов оплаты в 70+ странах
- Собственная платёжная платформа — миллиарды транзакций в год
- Поддержка: карты, PayPal, Apple Pay, Google Pay, наличные, Uber Cash, корпоративные аккаунты
- **Uber Money** — финансовые продукты, включая дебетовую карту и кошелёк

#### Система рейтингов и отзывов

- Двустороннее оценивание — пассажир оценивает водителя и наоборот, шкала 1–5
- Рейтинг рассчитывается как скользящее среднее за последние 500 поездок
- Минимальный порог для водителя — 4.6, варьируется по городам
- Возможность оставить комментарий и теги
- Комплименты и чаевые влияют на программу Uber Pro

#### Система уведомлений

- **Push** — Firebase Cloud Messaging для Android, APNs для iOS
- **SMS** — через Twilio и локальных провайдеров
- **In-app** — real-time через WebSocket
- **Email** — транзакционные чеки через собственную систему
- Персонализация уведомлений на основе ML

#### Система промокодов и лояльности

- **Uber One** — подписка за $9.99 в месяц: бесплатная доставка, скидки 5–10% на поездки
- Промокоды с ограничением по городу, времени, типу поездки
- Реферальная программа с бонусами за приглашённых
- Накопление баллов за поездки

#### Антифрод система

- ML-модели для обнаружения: фейковых GPS, мошеннических поездок, фрода с промокодами, скомпрометированных карт
- **Mastermind** — платформа правил для real-time решений
- Анализ поведенческих паттернов — velocity checks, device fingerprinting
- Обнаружение коллаборативного фрода — сговор водитель и пассажир

#### Real-time трекинг и геолокация

- Обработка миллионов GPS-обновлений в секунду
- Apache Kafka для потоковой обработки геоданных
- Ringpop для партиционирования данных по географии
- Анимация на клиенте с интерполяцией между точками
- Фильтр Калмана и map matching для повышения точности

#### Расчёт ETA

- **DeepETA** — ML-модель на основе Transformer-архитектуры
- Учитывает: трафик, погоду, исторические данные, события, время суток
- Средняя ошибка — около 2 минут

> **Источник:** [Uber DeepETA](https://eng.uber.com/deepeta-how-uber-predicts-arrival-times/)

#### Система поддержки клиентов

- In-app чат-бот с ML для автоматического решения типовых проблем
- Классификация обращений и маршрутизация к агентам
- Self-service: корректировка стоимости, потерянные вещи, возвраты

## 2. Яндекс Такси / Яндекс Go

**Страна:** Россия | **Охват:** Россия, СНГ, Израиль, Африка, Восточная Европа | **Основан:** 2011

### A. Архитектура системы

#### Тип архитектуры: Микросервисная — более 1000 микросервисов

Яндекс Такси — часть экосистемы Яндекса, использующая общую инфраструктуру и технологии. Архитектура эволюционировала от монолитного Python-приложения к распределённой системе на C++ и Python.

**Ключевые сервисы и их взаимодействие:**

- **Order Service — backend-cpp** — ядро системы управления заказами, написано на C++ для максимальной производительности
- **Dispatch — Lookup** — подбор водителей, использует геоиндекс и scoring-модели
- **Pricing Service** — расчёт стоимости поездки, surge-коэффициенты
- **Tracker** — отслеживание позиций водителей в реальном времени
- **Driver Protocol** — API для приложения водителя
- **Passenger Protocol** — API для приложения пассажира
- **Billing** — расчёты с водителями и партнёрами
- **Tariff Service** — управление тарифами по зонам и городам
- **Candidates** — поиск кандидатов-водителей по геоиндексу
- **Surge** — сервис динамического ценообразования
- **Userplaces** — сохранённые адреса пользователей
- **Geotracks** — хранение и анализ треков поездок

**Системы обмена сообщениями:**

- **YDB Topics — ранее Logbroker** — собственная система потоковой обработки событий, аналог Kafka
- **gRPC** — межсервисное взаимодействие
- **userver** — собственный асинхронный фреймворк для микросервисов на C++, open source
- **Yandex Message Queue** — очереди сообщений

**Подход к хранению данных:**

| Технология | Назначение |
|-----------|-----------|
| **YDB** | Собственная распределённая NewSQL БД — основное хранилище |
| **PostgreSQL** | Ряд сервисов, работающих с реляционными данными |
| **MongoDB** | Документоориентированное хранение для отдельных сервисов |
| **Redis** | Кеширование, сессии, временные данные |
| **ClickHouse** | Аналитика и метрики в реальном времени |
| **YT — YTsaurus** | MapReduce-система для batch-аналитики, собственный аналог Hadoop |
| **S3 — Yandex Object Storage** | Хранение файлов, документов, логов |

**Инфраструктура:**

- **Yandex Cloud** — собственное облако, все сервисы хостятся внутри экосистемы
- **Yandex Deploy — Nanny** — собственная система деплоя и оркестрации
- Docker-контейнеры в собственных дата-центрах
- **Golovan и Solomon** — системы мониторинга и алертинга
- **Arcanum** — собственная система code review
- **Arc** — собственная VCS, используемая вместо Git для монорепозитория

> **Источники:** [Яндекс Хабр](https://habr.com/ru/companies/yandex/articles/), [YDB Documentation](https://ydb.tech/), [userver](https://userver.tech/)

### B. Пользовательские модули

#### Приложение пассажира — Яндекс Go

Яндекс Go — суперприложение, объединяющее такси, доставку еды, продуктов, курьерскую доставку и другие сервисы.

**Основные экраны и потоки:**

1. **Главный экран**
   - Карта с текущим местоположением — Яндекс Карты
   - Поле ввода «Куда» с автокомплитом через Яндекс Геокодер
   - Виджеты сервисов: Такси, Еда, Лавка, Доставка, Драйв
   - Быстрые адреса — дом, работа
   - Промо-баннеры и акции
   - Прогноз погоды и курс валют

2. **Экран выбора маршрута**
   - Автокомплит через Яндекс Саджест
   - Выбор точки на карте
   - Множественные промежуточные точки
   - Избранные адреса
   - Поиск по организациям через Яндекс Справочник

3. **Экран выбора тарифа**
   - Эконом, Комфорт, Комфорт+, Бизнес, Минивэн, Детский, Грузовой
   - Цена для каждого тарифа с учётом surge
   - ETA подачи для каждого тарифа
   - Опции: детское кресло, кондиционер, тихая поездка, животное, курьерская доставка
   - «Хочу ехать» — запрос поездки

4. **Экран ожидания водителя**
   - Карта с перемещением водителя
   - Информация о водителе — фото, имя, рейтинг, автомобиль, номер, цвет

### C. Ключевые подсистемы

#### Алгоритм матчинга — аукцион

- **Нет классического матчинга** — главное отличие от всех конкурентов
- Пассажир создаёт заказ с предложенной ценой
- Водители видят заказ и могут: принять цену, предложить свою, пропустить
- Пассажир видит все предложения и выбирает водителя
- По сути это **reverse auction** — обратный аукцион
- Sorting: предложения сортируются по цене, ETA, рейтинг у водителя

#### Система рекомендации цены

- ML-модель анализирует: расстояние, время, пробки, погоду, спрос, исторические данные
- Рекомендуемая цена — не обязательная, а ориентировочная
- Пассажир может указать цену выше или ниже рекомендуемой
- Алгоритм обучается на реальных сделках — какие цены принимаются чаще

#### Маршрутизация

- Google Maps API для маршрутов
- Расчёт расстояния и предполагаемого времени поездки для рекомендации цены

- Банковские карты

- Apple Pay, Google Pay
- Собственная платёжная система inDriver для онлайн-оплаты
- В ряде стран — локальные методы

#### Система рейтингов

- Двустороннее оценивание — 1–5 звёзд
- Рейтинг виден при торге — пассажир видит рейтинг водителя, водитель видит рейтинг пассажира
- Высокий рейтинг — конкурентное преимущество при торговле

#### Антифрод

- GPS validation
- Обнаружение фейковых аккаунтов
- Мониторинг подозрительных ценовых паттернов
- Device fingerprinting

#### Real-time трекинг

- Стандартное GPS-трекинг через WebSocket
- Kafka для обработки потоков
- Обновление каждые 5 секунд
