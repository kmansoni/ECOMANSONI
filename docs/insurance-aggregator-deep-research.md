### 3.8 Визуальная организация ОСАГО-формы

**InsSmart**: Одна длинная форма с секциями (компактная, минимум полей)
- Секции: Данные полиса → ТС → Владелец → Водители → Рассчитать

**Pampadu**: Вертикальная форма, больше полей (паспорт, адрес)
- Секции: ТС → Собственник → Страхователь → Водители → Рассчитать

**Sravni Labs**: Полная форма со всеми данными, tabs наверху
- Секции: Общие данные → Данные ТС → Собственник → Страхователь → Водители → Данные полиса → Рассчитать
- Доп: «Сохранить черновик», «Очистить»

**Polis.Online**: 7-шаговый визард (самая структурированная)
- Шаги: 1. Данные ТС → 2. Страхователь → 3. Собственник → 4. Водители → 5. Данные полиса → 6. Контакты → 7. Финальные условия (согласия)
- Доп: Предупреждение про ограничение по номеру телефона, правовой текст 152-ФЗ

## 4. КАСКО — детальное сравнение {#4-каско}

### 4.1 Доступность

| Платформа | Тип | Подход |
|---|---|---|
| **InsSmart** | Полный калькулятор | Нативная форма, расчёт от нескольких СК |
| **Pampadu** | Через менеджера + Мини-КАСКО | Заявка менеджеру (онлайн не считает), Мини-КАСКО отдельно |
| **Sravni Labs** | Полный калькулятор + через менеджера | Нативная 3-шаговая форма (Т-Страхование онлайн) + заявка менеджеру |
| **Polis.Online** | В списке продуктов | Доступен, но детали формы не изучены |

### 4.2 Sravni Labs КАСКО — самая детальная форма (образцовая)

**3 основных таба:**
- Оформить полис | Полисы и документы | Расчёты

**Подтабы оформления:**
- Онлайн оформление Т-страхование | Заявка через менеджера | Мои заявки | Черновики | Мои расчеты

**3-шаговый визард**: Ввод данных → Расчёт → Оформление

**Поля формы КАСКО (Sravni Labs):**

| Секция | Поля |
|---|---|
| **Общие данные** | Срок (1 год), Даты начала/окончания, Покрытия и риски, Вариант ремонта (Официальный дилер), Ограничение пробега, Стоимость авто ₽ |
| **Автомобиль** | Рег.номер / VIN (автозаполнение), Марка, Модель, Год, Мощность, Объём двигателя, Тип кузова, Тип КПП, Пробег км, VIN, СТС серия/номер/дата, Рег.номер |
| **Доп.сведения авто** | Кол-во ключей, Срок владения, Кол-во страх.случаев за год, Праворульный?, Менялись номера?, Автозапуск?, В кредит? |
| **Страхователь** | Фамилия, Имя, Отчество, ДР, Паспорт (тип/серия/номер/код подразделения/дата), Адрес + квартира, Проживание = регистрация?, Телефон, E-mail |
| **Водители** | ФИО, ДР, ВУ серия/номер/дата выдачи/дата первого ВУ, Менял ВУ?, «+ Добавить водителя» |
| **Действия** | Рассчитать, Сохранить черновик, Очистить |

### 11.2 Подход к формам

**Подход A: Длинная форма (InsSmart, Pampadu, Sravni Labs)**
- Все поля на одной странице
- Секции визуально разделены заголовками
- Одна кнопка «Рассчитать» внизу
- Плюс: все данные видны сразу
- Минус: intimidating для новичков

**Подход B: Step-by-step wizard (Polis.Online)**
- Каждый шаг = один тип данных
- Прогресс-бар / навигация по шагам
- Плюс: не пугает, можно сохранить прогресс
- Минус: больше кликов

**Подход C: Iframe-виджет (Sravni Labs для некоторых продуктов)**
- Встраиваемый калькулятор от третьей стороны
- Плюс: zero-code, мгновенный запуск
- Минус: нет контроля UX, зависимость от провайдера

### 11.3 Архитектурные решения (выводы)

1. **Главная модель — B2B SaaS для агентов**. Потребительский (B2C) фасад — вторичен
2. **Sidebar + Cards** — стандартный UI-паттерн для всех платформ
3. **Продукты = карточки** на главной, каждый ведёт на калькулятор
4. **CRM обязателен** — без него агенты не придут (InsSmart лидер тут)
5. **Финансы / Баланс** — обязательный модуль (агент должен видеть и выводить заработок)
6. **Реферальная программа** — есть у всех в том или ином виде
7. **Черновики** — важная фича (Sravni, Polis.Online), позволяет не потерять частично заполненный расчёт

### 13.3 Форма ОСАГО — GOLDEN STANDARD

Основываясь на сравнении 4 платформ, оптимальная форма ОСАГО:

```
Шаг 1: Данные ТС
  - Госномер (автозаполнение: марка, модель, год, мощность)
  - «Госномер отсутствует» checkbox
  - Категория ТС (B default)
  - Цель использования (Личная default)
  - VIN
  - Тип документа (ПТС/СТС), серия, номер, дата выдачи

Шаг 2: Собственник
  - Тип (ФЛ/ИП/ЮЛ)
  - ФИО
  - Дата рождения
  - Паспорт серия/номер, дата выдачи
  - Адрес регистрации

Шаг 3: Страхователь
  - «Является собственником» checkbox → skip
  - Те же поля что у собственника

Шаг 4: Водители
  - «Без ограничений» (мультидрайв) toggle
  - Список водителей:
    - ФИО
    - Дата рождения
    - ВУ серия/номер, дата выдачи
    - Дата начала стажа
    - «Предыдущее ВУ для переноса КБМ» checkbox
    - «Скопировать из собственника/страхователя» кнопка
  - «+ Ещё один водитель» кнопка

Шаг 5: Данные полиса
  - Дата начала
  - Срок (12 месяцев default)
  - «Зелёный коридор» checkbox (ввод номера предыдущего полиса)

Шаг 6: Контакты
  - Телефон
  - E-mail

Шаг 7: Подтверждение
  - Согласие на обработку ПД (152-ФЗ)
  - Согласие на передачу данных СК
  - Подтверждение достоверности
  - [Кнопки: «Сохранить черновик» | «Рассчитать»]
```

### 13.4 Ключевые архитектурные решения

1. **Role-based architecture**: Пользователь, Агент, Админ — разные интерфейсы и права
2. **Adapter pattern для СК**: Каждая страховая = отдельный адаптер с единым интерфейсом
3. **Quote Session**: Сессия расчёта с кешированием (TTL 15 мин)
4. **Draft system**: Автосохранение черновиков незавершённых расчётов
5. **Commission engine**: Расчёт комиссии агента для каждого предложения
6. **Loyalty engine**: Уровни агента, квартальный расчёт, прогрессивная комиссия
7. **Referral engine**: Генерация ссылок, отслеживание активаций/расчётов/оформлений
8. **Push/notification**: Напоминания о пролонгации, ДР клиентов, истечение полисов
