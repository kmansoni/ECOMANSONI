# Часть IV — Дизайн-Система и UX

---

## 1. Философия и Принципы Дизайна

### 1.1 Основные принципы

**1. Clarity over Cleverness (Ясность важнее умности)**
Каждый элемент должен быть понятен с первого взгляда. Нет скрытых функций, нет загадочных иконок без подписей в критических местах. Пользователь не должен думать — он должен действовать.

**2. Speed is a Feature (Скорость — это функция)**
Каждая миллисекунда задержки снижает конверсию. Skeleton-экраны вместо спиннеров, optimistic UI (показываем результат до ответа сервера с откатом при ошибке), instant feedback на каждое действие.

**3. Mobile-First, Desktop-Enhanced**
62% пользователей NEXUS будут использовать мобильные устройства. Дизайн начинается с 375px. Десктоп добавляет, а не упрощает.

**4. Progressive Disclosure (Прогрессивное раскрытие)**
Показываем только то, что нужно прямо сейчас. Детали — по требованию. Упрощённый checkout для повторных покупателей, расширенный для первой покупки.

**5. Honest Design (Честный дизайн)**
Нет тёмных паттернов: нет скрытых подписок, нет pre-checked boxes, нет искусственных таймеров "только сейчас!". NEXUS строит доверие, а не давит.

**6. Inclusive by Default (Инклюзивность как умолчание)**
WCAG 2.1 AA — не опция, а требование. Контрастность, размеры кнопок, навигация с клавиатуры, screen reader поддержка.

### 1.2 Эмоциональный дизайн

**Персонажность NEXUS:** уверенный, современный, дружелюбный, прямолинейный.
- **Не:** агрессивный, манипулятивный, перегруженный
- **Да:** чистый, функциональный, с характером

### 2.6 Анимации

```css
/* Длительности */
--duration-instant:    50ms;
--duration-fast:       100ms;
--duration-normal:     200ms;
--duration-slow:       300ms;
--duration-slower:     500ms;

--ease-default:     cubic-bezier(0.4, 0, 0.2, 1);   /* Material Design standard */
```

**Принципы анимаций:**
- Добавить в корзину: scale(1.1) → scale(1) + fly-to-cart анимация иконки
- Hover на карточку товара: translateY(-2px) + shadow elevation
- Skeleton → контент: fade in 300ms
- Modal открытие: slide-up на мобайл, scale+fade на десктоп
- Уведомление: slide-in справа, auto-dismiss с прогресс-баром

## 4. Детальные User Flows (15 ключевых сценариев)

### Flow 1: Первая регистрация покупателя

```
Способ входа →
  ├── По телефону (основной):
  │   Главная → "Войти/Зарегистрироваться" →
  │   Ввод номера [+7 (___) ___-__-__] →
  │   [Получить код] →
  │   Ввод 4-значного OTP (auto-submit) →
  │   НОВЫЙ ПОЛЬЗОВАТЕЛЬ: экран "Как вас зовут?" [Имя] + [Город] →
  │   [Начать покупки!] →
  │   Главная с онбординг-тостом "Добро пожаловать! Вот ₽500 на первую покупку"
  │
  └── Через Google/VK/Yandex:
      Главная → Кнопка OAuth → Redirect → Callback →
      Если новый: экран имени/города →
      Главная с приветствием

Время: < 60 секунд (для нового пользователя)
Ошибки:
  - Неверный OTP: "Неверный код. Осталось 4 попытки"
  - Истёк OTP: "Код устарел. Запросить новый"
  - Блокировка: "Слишком много попыток. Попробуйте через 15 минут"
```

### Flow 2: Первая покупка

```
Поиск → Поисковая строка "наушники беспроводные" →
Результаты поиска (SERP) → Фильтрация: цена до ₽5000, бренд Samsung →
Клик на карточку →
PDP (карточка товара):
  - Просмотр фото (swipe gallery)
  - Выбор цвета: [Чёрный] → обновление фото
  - Чтение отзывов (скролл)
  - "Добавить в корзину" →
  - Пуш-анимация в корзину
Корзина:
  - Проверка товаров
  - "Оформить заказ" →
Checkout:
  - Шаг 1: Добавить адрес → автодополнение → [Сохранить адрес]
  - Выбор доставки: [NEXUS Express сегодня до 21:00 — ₽490] ✓
  - Шаг 2: Оплата → [Карта] → ввод данных карты
  - [Сохранить карту]
  - Шаг 3: Подтверждение → [Оформить заказ ₽4 689]
  - 3DS экран банка →
Экран успеха:
  - "Заказ №NX-123456 оформлен!"
  - "Ожидайте сегодня до 21:00"
  - Кнопка "Отследить заказ"
  - Рекомендации "Добавьте к наушникам"

Время: первая покупка ~ 4-7 минут (с регистрацией)
Повторная покупка (сохр. адрес + карта): ~ 90 секунд
```

### Flow 3: Поиск по фото

```
Главная / Поиск → Иконка камеры в поисковой строке →
Запрос разрешения камеры (первый раз) →
Экран камеры:
  - Режимы: [Камера] / [Галерея] / [Ссылка на фото]
  - Подсказка: "Сфотографируйте товар, который хотите найти"
  - Кнопка съёмки →
Обработка (spinner < 3 сек) →
Результаты:
  - "Найдено: похожие товары" [grid]
  - Confidence: "Мы думаем, это: Samsung Galaxy Watch" (если определено)
  - Кнопка "Уточнить запрос"
  - Если нет результатов: "Не нашли похожее. Попробуйте описать словами"
```

### Flow 4: Оформление возврата

```
Профиль → Мои заказы → Заказ NX-123456 →
[Вернуть товар] →

Шаг 1: Выбор товара из заказа (если несколько)
Шаг 2: Причина возврата:
  [Не подошёл размер/фасон] ← без фото
  [Товар пришёл повреждённый] ← требует фото
  [Не соответствует описанию] ← требует фото
  [Другое] ← текстовое поле

Шаг 3 (если требуется фото):
  - Кнопка "Добавить фото(1-5)" → камера / галерея

Шаг 4: Выбор способа возврата:
  [🚴 Курьер заберёт] бесплатно (Premium) / ₽190
  [📦 Сдать в ПВЗ] бесплатно → карта ближайших
  [📮 Почтой России] вам возместят ₽100

Шаг 5: Подтверждение:
  - "Возврат: Samsung Galaxy Buds Pro, Чёрный"
  - "Причина: Не подошёл"
  - "Способ: Курьер, 15–17 января"
  - "Возврат средств на карту *4567 в течение 3 дней"
  - [Подтвердить возврат]

Трекинг возврата:
  REQUESTED → COURIER_ASSIGNED → PICKED_UP → IN_TRANSIT → RECEIVED → REFUNDED

SMS/Push уведомления на каждый статус
```

### Flow 5: Регистрация продавца

```
Главная → "Продавать на NEXUS" (footer / landing) →
Лендинг продавцов:
  - Преимущества (UVP для продавцов)
  - Тарифы и комиссии
  - Истории успеха
  - [Начать продавать]

Шаг 1: Тип аккаунта
  [Физическое лицо / ИП] → Standard KYC
  [ООО / АО] → Business KYC
  
Шаг 2: Основная информация
  - ФИО / Название организации
  - Email (для уведомлений)
  - Телефон (OTP верификация)

Шаг 3: ИНН + автозаполнение из ФНС API
  - ИНН → система автоматически получает: наименование, статус, адрес
  - Подтверждение данных

Шаг 4: Документы
  - Загрузка паспорта (главный разворот)
  - Selfie с паспортом (liveness)
  - Для ООО: ОГРН + устав (PDF)

Шаг 5: Банковские реквизиты
  - БИК + расчётный счёт (автоверификация ЦБ)
  - Или привязка карты

Шаг 6: Договор-оферта (PDF превью + checkbox + дата)

Результат: "Заявка отправлена. Проверка 1-24 часа"
Email + SMS: "Мы проверяем ваши данные..."

При одобрении:
Email: "Ваш аккаунт продавца активирован! Добавьте первый товар"
→ redirect в Seller Center Onboarding Wizard
```

### Flow 6: Добавление первого товара (продавец)

```
Seller Center → Товары → [+ Добавить товар]

Wizard добавления товара:

Шаг 1: Выбор категории
  - Поиск по категории: "наушники" → предложения
  - Или навигация по дереву
  - Выбор "Электроника → Аудио → Наушники и гарнитуры"

Шаг 2: Основная информация
  - Название товара (подсказки от AI: "Samsung Galaxy Buds2 Pro черные беспроводные")
  - Бренд (из справочника + "Другой бренд")
  - Артикул (sku-123)
  - Штрих-код (EAN-13, необязательно)

Шаг 3: Фотографии
  - Drag & drop зона + кнопка
  - Минимум 3, рекомендуется 7+
  - Автоматическое сжатие и оптимизация
  - AI проверка: "Фото слишком тёмное, поднимите освещение"
  - AI заполнение: определяет цвет, состояние товара из фото

```

```
  - Крупный текст (18pt+): минимум 3:1

  - Primary button (#2563EB на белом): 8.59:1 ✅
  - Body text (#374151 на белом): 9.71:1 ✅
  - Secondary text (#6B7280 на белом): 4.64:1 ✅

  - Кнопки навигации: 48×48 px минимум

  - Tab order логический (слева направо, сверху вниз)
```

## 8. A/B Тестирование — План на 6 месяцев

| Месяц | Эксперимент | Гипотеза | Метрика |
|-------|------------|---------|---------|
| 1 | CTA button text: "Добавить в корзину" vs "Купить" | Прямой призыв повысит CVR | Add-to-cart rate |
| 1 | OTP: 4 цифры vs 6 цифр | 4 цифры быстрее → меньше отвала | Registration completion rate |
| 2 | Количество фото на PDP: 5 vs 10 главный показ | Больше фото → выше CVR | PDP→Cart CVR |
| 2 | Social proof: "1 234 купили" vs без него | Социальное доказательство → CVR | PDP CVR |
| 3 | Место кнопки "Купить": fixed bottom vs in-content | Sticky кнопка → выше CVR мобайл | Mobile CVR |
| 3 | Бейдж "Verified Review" на отзывах | Блокчейн бейдж → доверие → CVR | Conversion lift |
| 4 | Принудительный onboarding profile vs skip | Заполненный профиль → лучше рекомендации → retention | Day-7 retention |
| 4 | Price anchoring: показ "Средняя цена в категории" | Anchor → выше perceived value | CVR expensive items |
| 5 | Email: персонализированная тема vs generic | Персонал. → open rate | Email open rate |
| 5 | Checkout: 1 страница vs 3 шага | Одна страница → меньше отвала | Checkout completion |
| 6 | Live стрим блок на главной: топ vs низ | Выше → больше просмотров | Live stream starts |
| 6 | Push частота: 3/день vs 1/день | Меньше → меньше оттока | Push opt-out rate |

**Методология A/B тестирования:**
- Инструмент: GrowthBook (open source, self-hosted)
- Распределение: 50/50 случайное на user_id
- Минимальная длительность: 14 дней (исключить day-of-week эффекты)
- Статистическая значимость: 95% (p < 0.05)
- Минимальный детектируемый эффект: +3% для конверсионных метрик
- Guardrail метрики: revenue per user (не должен падать при оптимизации частных метрик)
