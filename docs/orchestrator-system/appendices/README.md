## Архитектурные решения (ADR)

### ADR-001: Трёхуровневая память vs единое векторное хранилище

**Дата:** 2026-01-15  
**Статус:** Принято

**Контекст:**  
Нужна персистентная память для AI агента. Альтернативы: (1) единый Vector Store, (2) трёхуровневая система.

**Решение:**  
Трёхуровневая система (Working Memory + Episodic Memory + Semantic Memory).

**Обоснование:**
- Working Memory даёт быстрый доступ к текущей сессии без embedding overhead
- Episodic Memory хранит структурированные эпизоды с метаданными (importance, topics)
- Semantic Memory организует факты по темам с confidence scoring
- Разные уровни имеют разные стратегии поиска: window-based, TF-IDF, topic-based

**Trade-offs:**
- ➕ Быстрее и дешевле, чем embedding всего
- ➕ Структурированные данные легче фильтровать
- ➖ Нет полноценного семантического поиска (нет embedding)
- ➖ Три компонента вместо одного

### ADR-002: FastAPI + WebSocket vs GraphQL Subscriptions

**Дата:** 2026-01-20  
**Статус:** Принято

**Контекст:**  
Нужен потоковый API для streaming ответов агента в реальном времени.

**Решение:**  
FastAPI с WebSocket для потоков + REST для операций с ресурсами.

**Обоснование:**
- WebSocket нативен для потоковых ответов, низкая latency
- FastAPI — наилучшая производительность среди Python web frameworks
- GraphQL Subscriptions добавляет сложность без значимых преимуществ для данного use case

### ADR-003: ReAct паттерн для агентов

**Дата:** 2026-01-25  
**Статус:** Принято

**Контекст:**  
Выбор архитектуры агентов: ReAct, Plan-and-Execute, Reflexion, AutoGPT.

**Решение:**  
ReAct (Reasoning + Acting) с отдельной Planning фазой для сложных задач.

**Обоснование:**
- ReAct доказал эффективность на широком спектре задач
- Прозрачность: chain-of-thought видна пользователю
- Легко имплементировать с любым LLM провайдером
- Plan-and-Execute добавлен поверх для multi-step задач

### ADR-004: MCP для внешних инструментов vs кастомный плагин API

**Дата:** 2026-02-01  
**Статус:** Принято

**Контекст:**  
Нужен стандартный способ подключения внешних инструментов (браузер, БД, API).

**Решение:**  
Model Context Protocol (MCP) как стандарт для всех внешних инструментов.

**Обоснование:**
- Открытый стандарт с растущей экосистемой готовых серверов
- Изоляция: каждый инструмент — отдельный процесс
- Безопасность: sandbox через stdio транспорт
- Не нужно писать кастомные интеграции

---

### ADR-005: NVIDIA Riva для STT/TTS vs cloud провайдеры

**Дата:** 2026-02-10  
**Статус:** Принято

**Контекст:**  
Голосовой интерфейс ARIA требует высококачественного STT/TTS с низкой latency.

**Решение:**  
NVIDIA Riva как основной провайдер с fallback на cloud APIs.

**Обоснование:**
- Локальный деплоймент: нет зависимости от внешних сервисов
- Задержка < 300 мс vs 500–1000 мс у cloud
- Полный контроль данных (важно для enterprise)
- Riva поддерживает качественный русский язык

## FAQ

**Q: Как агент получает доступ к моему коду?**  
A: Через VS Code Extension API — только к файлам в текущем workspace. Агент не может читать файлы за пределами проекта без явного разрешения через MCP filesystem сервер.

**Q: Где хранятся данные памяти?**  
A: В JSON файлах на вашем компьютере (по умолчанию `/tmp/ai_memory`). Данные не передаются третьим сторонам, только в LLM API провайдер (OpenAI/Anthropic) при генерации ответов.

**Q: Что происходит с памятью при переустановке?**  
A: Данные сохраняются в `storage_path`. При переустановке укажите тот же путь — данные восстановятся автоматически.

**Q: Могу ли я увидеть, что агент собирается сделать перед выполнением?**  
A: Да. Для всех операций с файлами показывается diff-view в VS Code. Команды в терминале требуют явного подтверждения. Это поведение настраивается через `aiCompanion.requireApprovalForFileChanges`.

**Q: Агент медленно отвечает. Почему?**  
A: Research Phase может занимать 10–30 секунд при анализе большого проекта. Для ускорения: (1) укажите конкретный файл в запросе, (2) уменьшите `max_files_in_research` в конфигурации.

**Q: Как работает ARIA без GPU?**  
A: Система автоматически переключается на Google Cloud STT/TTS или Azure Cognitive Services. Качество незначительно хуже, latency выше (~500 мс). Настраивается через `ARIA_FALLBACK_PROVIDER`.

**Q: Можно ли использовать локальную LLM модель?**  
A: Да. Установить `LLM_PROVIDER=ollama`, `OLLAMA_BASE_URL=http://localhost:11434`. Протестировано с Llama 3.1, Mistral, Qwen2.5-Coder.

**Q: Как сбросить память агента?**  
A: `DELETE /api/v1/memory/all` или вручную удалить JSON файлы в `storage_path`. Внимание: это необратимая операция.
