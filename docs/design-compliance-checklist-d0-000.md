# D0.000 — Design Compliance Checklist (Attach to every UI Story)

Цель: единый чеклист, который **обязан** проходить каждый UI-элемент/состояние/экран во всех фазах.

Правило процесса:
- Этот чеклист прикладывается к каждой Story с UI.
- Story не может быть “Done”, пока все пункты не отмечены.

---

## A) Tokens & Components
- Используются только существующие UI компоненты проекта (не вводить новые библиотеки/темы).
- Используются только существующие Tailwind tokens/переменные темы.
- Нет хардкода новых цветов/шрифтов/теней.

## B) Layout & Safe Area
- Все полноэкранные поверхности учитывают safe-area.
- Нет “сломанных” 100dvh/scroll контейнеров.
- Fixed элементы не перекрывают контент.

## C) Mode Flags (единый движок)
- Reels/Story/Post/Live — один контейнер/движок, разные mode flags.
- Нет трёх отдельных камер/трёх отдельных upload flows.

## D) Error/Degrade UX
- Нет системных alert/prompt без стилизации.
- Любая ошибка имеет:
  - краткий текст
  - категорию (error_code)
  - next-action (retry/change/settings)
- Нет “вечных спиннеров”: всегда есть статус.

## E) Consistency
- Отступы/радиусы/типографика совпадают с существующими экранами.
- Модалки и sheets используют текущие glass‑паттерны.
- Иконки — из текущего набора.

## F) Accessibility (минимум)
- Кнопки имеют aria-label где нужно.
- Контраст читаемый.
- Focus states не сломаны.

## G) Performance
- Нет лишних heavy анимаций.
- Нет утечек (URL.createObjectURL — revoke и т.д.).
