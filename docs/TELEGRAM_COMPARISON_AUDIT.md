# 📊 Полный аудит проекта «your-ai-companion» vs Telegram

## Дата аудита: 2026-03-04

## Executive Summary

Проведён комплексный анализ проекта по всем параметрам в сравнении с Telegram:
- **Инвентаризация**: ~60 страниц, ~200+ компонентов, ~100 хуков, 11 серверных микросервисов
- **Функции чата**: 68 проверенных функций — 42 реализованы (62%), 9 частично (13%), 17 отсутствуют (25%)
- **UI-элементы**: 26+ модалок/шитов, 14 отсутствующих UI-паттернов Telegram
- **Безопасность**: 6 критических, 7 высоких, 7 средних, 4 низких уязвимостей
- **Общая оценка покрытия Telegram-функций**: ~75%

## 3. БЕЗОПАСНОСТЬ — УЯЗВИМОСТИ

### 3.1 🔴 Критические (P0) — исправить немедленно

| # | Уязвимость | Файл | Описание | Рекомендация |
|---|-----------|------|----------|-------------|
| C-1 | Токены в localStorage без шифрования | `src/auth/sessionStore.ts` | access_token/refresh_token в открытом виде. XSS = полный доступ | Перейти на HttpOnly Secure SameSite=Strict cookies |
| C-2 | Device secret в localStorage | `src/auth/deviceIdentity.ts` | device_uid + device_secret в JSON. XSS = компрометация identity | Шифровать через Web Crypto перед сохранением |
| C-3 | OTP store в памяти (Map) | `server/phone-auth/index.mjs` | При рестарте все OTP теряются. Не масштабируется | Перевести на Redis с TTL |
| C-4 | Math.random() для OTP | `server/phone-auth/index.mjs:248` | Не криптографически безопасен | Заменить на crypto.randomInt(100000, 999999) |
| C-5 | Fail-open в trust enforcement | `server/trust-enforcement/middleware.ts` | При ошибке сервиса — middleware пропускает запрос | Возвращать 503, а не next() |
| C-6 | account_id: "unknown" в JWT | `services/auth/src/index.ts:162` | При refresh token ротации JWT без привязки к аккаунту | Извлекать account_id из сессии БД |

### 3.2 🟠 Высокие (P1)

| # | Уязвимость | Файл | Описание |
|---|-----------|------|----------|
| H-1 | Нет Double Ratchet | `src/lib/e2ee/crypto.ts` | PFS только через ручную ротацию. Компрометация ключа раскрывает все сообщения |
| H-2 | Нет login notifications | — | Пользователь не узнает о входе с нового устройства |
| H-3 | Redis race condition | `server/trust-enforcement/rate-limiter.service.ts:163` | fetch→compute→set без атомарности |
| H-4 | NSFW проверка только на клиенте | `src/lib/moderation/imageFilter.ts` | Модифицированный клиент обходит проверку |
| H-5 | Текстовый фильтр легко обходим | `src/lib/moderation/textFilter.ts` | lowerText.includes(word) — обходится юникодом |
| H-6 | JWT 7 дней без ротации | `server/phone-auth/index.mjs:258` | Длинный lifetime в legacy endpoint |
| H-7 | Cloud password как SHA-256 без salt | `src/lib/passcode.ts` | Уязвим к rainbow tables. Нужен Argon2id/PBKDF2+salt |

### 3.3 🟡 Средние (P2)

| # | Уязвимость | Описание |
|---|-----------|----------|
| M-1 | Нет CSRF токенов | CORS + SameSite, но нет явных CSRF tokens |
| M-2 | PublicKey cache TTL 5 мин | Кешированный ключ может быть устаревшим при компрометации |
| M-3 | NonceManager ограничен 10K записей | Replay после eviction возможен |
| M-4 | Memory fallback для KeyStore | При недоступности IndexedDB ключи теряются |
| M-5 | Нет At-Rest шифрования | Зависит от конфигурации Supabase хостинга |
| M-6 | Удаление аккаунта не реализовано | GDPR/Закон о ПДн требует возможность удаления |
| M-7 | Self-destruct media отсутствует | Нет таймеров самоуничтожения для медиа |

### 3.4 🟢 Низкие (P3)

| # | Уязвимость | Описание |
|---|-----------|----------|
| L-1 | Нет rate limit на disableEncryption | Многократное включение/выключение E2EE |
| L-2 | Report система не найдена | Нет механизма жалоб |
| L-3 | Нет IP-reputation filtering | Trust scoring без IP-репутации |
| L-4 | Insertable Streams — Chrome-only | Firefox/Safari поддержка — TODO |
