# Dev Browser Hidden

Запускает dev сервер проекта и браузер с расширенными настройками конфиденциальности для локального тестирования.

## Когда использовать

- Локальная разработка с дополнительной изоляцией
- Тестирование без утечки реальных данных браузера
- Отладка без влияния расширений браузера
- Запуск нескольких изолированных сессий

## Что делает этот скил

1. **Запускает dev сервер**: Определяет тип проекта и запускает соответствующий dev сервер
2. **Настраивает браузер**: Запускает Chrome с флагами для дополнительной изоляции:
   - Отключение WebRTC IP утечек
   - Блокировка third-party cookies
   - Отключение geolocation API
   - Использование свежей сессии без расширений
   - Блокировка сторонних запросов

## Использование

### Запуск с настройками по умолчанию

```
Запусти dev сервер и браузер скрытый
```

### Указать порт

```
Запусти dev сервер на порту 3001 и браузер скрытый
```

### Только браузер (если сервер уже запущен)

```
Запусти только браузер скрытый
```

## Флаги браузера

- `--new-window` - новое окно без профиля
- `--disable-webrtc` - отключение WebRTC
- `--disable-geolocation` - блокировка геолокации
- `--disable-third-party-cookies` - блокировка third-party cookies
- `--incognito` - режим инкогнито
- `--disable-extensions` - без расширений

## Примеры

**Запуск для Next.js проекта:**

```

**Результат:**

```

```

```

## Поддерживаемые фреймворки

- Next.js (`npm run dev`)
- Vite (`npm run dev`)
- CRA (`npm start`)
- Vue CLI (`npm run serve`)
- Nuxt (`npm run dev`)
- Remix (`npm run dev`)

## Ограничения

- Только Windows
- Требует Google Chrome
- Некоторые сайты могут блокировать доступ с локальных IP

## Безопасность

Этот скил предназначен исключительно для локальной разработки и тестирования. 
Не используйте для обхода защиты на production сайтах.
